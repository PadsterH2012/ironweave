# Project Details & Features Tabs — Design

> **Date:** 2026-03-16
> **Status:** Approved
> **Goal:** Give every project a living record of intent vs reality (Details tab) and a full feature lifecycle tracker (Features tab), so ideas are never lost and implementation status is always visible.
> **Motivation:** ADHD-friendly project management — capture ideas immediately, park without guilt, return safely months later, see exactly where things stand.

---

## Decisions

- **Features as first-class entities** — new `features` and `feature_tasks` tables, separate from issues
- **Implement button creates Ironweave issues** — all work flows through the orchestrator (dogfooding)
- **Smart nudge for parked features** — surface related parked features via keyword overlap when agents work on similar tasks
- **PRD import accepts any text** — pasted markdown, raw text, conversation notes — extracted into features + tasks
- **Details is a versioned document** — diffed against previous version, removal protection, version history
- **Reality scan by agent** — reads code + docs, produces structured summary, insights feed knowledge graph
- **Auto-promote with verification gate** — features auto-move to "implemented" when tasks done, manual/test verification to "verified"
- **Cross-project dashboard summary** — features live in projects but dashboard shows global counts

## 2. Feature Lifecycle

### Status flow

```
💭 idea ──→ 📋 designed ──→ 🚧 in_progress ──→ ✅ implemented ──→ 🔍 verified
   │              │                │                    │
   └──→ ⏸️ parked ←──────────────────────────────────────┘
   │
   └──→ 🗑️ abandoned (with reason, never deleted)
```

### Auto-promotion rules (in orchestrator sweep)

1. **designed → in_progress**: First feature_task gets an issue_id
2. **in_progress → implemented**: All feature_tasks are done or skipped (none todo)
3. **implemented → verified**: Manual click or linked e2e test passes

### Smart nudge for parked features

On `spawn_team_agent()`:
- Extract keywords from the issue being worked on
- Query parked features with keyword overlap > 0.5
- Inject into agent prompt: "Related parked feature: {title}"
- Log to loom for visibility

### Abandoned features

Never deleted. Status set to "abandoned" with a reason. Preserved as a record.

## 3. Details Tab — Versioned Documents

### Left: Intent

- Editable markdown textarea with preview toggle
- On save: stores previous content, increments version, diffs for removal detection
- Removal warning: "The following was removed — was this intentional?" with restore option

### Right: Reality

- Read-only, generated by agent scan
- Agent reads source files, README, docs, config → produces structured summary
- Key insights pushed to knowledge graph as context patterns
- "Rescan" button triggers on demand

### Gap Analysis

- Keyword comparison between intent and reality documents
- Missing from code (red) — mentioned in intent, not found in reality
- Undocumented (amber) — found in reality, not mentioned in intent
- "Create Feature" action on missing items

## 4. API Endpoints

### Features

| Method | Path | Purpose |
|--------|------|---------|
| GET | `/api/projects/{pid}/features` | List (filterable by status) |
| GET | `/api/projects/{pid}/features/{id}` | Get with tasks |
| POST | `/api/projects/{pid}/features` | Create |
| PUT | `/api/projects/{pid}/features/{id}` | Update |
| DELETE | `/api/projects/{pid}/features/{id}` | Soft delete → abandoned |
| POST | `/api/projects/{pid}/features/{id}/park` | Park with reason |
| POST | `/api/projects/{pid}/features/{id}/verify` | Mark verified |
| POST | `/api/projects/{pid}/features/import` | Import PRD text |

### Feature Tasks

| Method | Path | Purpose |
|--------|------|---------|
| GET | `/api/features/{fid}/tasks` | List tasks |
| POST | `/api/features/{fid}/tasks` | Add task |
| PUT | `/api/features/{fid}/tasks/{id}` | Update task |
| DELETE | `/api/features/{fid}/tasks/{id}` | Remove task |

| Method | Path | Purpose |
|--------|------|---------|
| GET | `/api/features/summary` | Cross-project counts by status |

## 5. Frontend

### Features Tab (`FeaturePanel.svelte`)

- Status filter tabs: All | Ideas | Designed | In Progress | Implemented | Verified | Parked
- "Add Feature" + "Import PRD" buttons
- Expandable feature cards: status badge, title, task progress bar, description, PRD, tasks with "Implement" buttons, action buttons (Park/Verify/Abandon)
- Import PRD modal: paste any text, preview extracted features, confirm

### Details Tab (`ProjectDetailsPanel.svelte`)

- Split layout: Intent (editable, versioned) | Reality (read-only, generated)
- Removal detection with restore option
- Gap analysis at bottom: missing (red) | undocumented (amber)
- "Rescan" button for reality refresh

### Dashboard Widget

- Feature summary card: "3 ideas · 2 in progress · 5 implemented · 1 awaiting verification · 2 parked"
- Click through to projects
